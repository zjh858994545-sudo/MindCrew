# MindCrew 模拟企业服务台知识

这份文档用于解释 `service-desk-loop-schema.sql` 中的模拟工单为什么能代表真实业务场景。内容是合成数据，不包含任何真实公司内部信息。

## HR-Handbook / Leave-SOP

年假申请规则：

- 年假额度按入职日期折算，系统额度以 OA 展示为准。
- 试用期员工可以申请已折算产生的年假。
- 连续 2 天以内年假审批链路为直属上级、部门负责人。
- 超过 2 天或涉及关键发布窗口，需要 HRBP 补充确认。
- 请假前需要同步交接事项、项目影响和紧急联系人。

## IT-Account-SOP / VPN-MFA-Runbook

VPN 与 MFA 处理规则：

- 新设备登录失败时，优先检查本机时间同步和 MFA 绑定状态。
- MFA 失败可以在账号中心自助重绑。
- 自助重绑失败时提交 IT 工单，附员工工号、设备编号、报错截图。
- 管理后台访问属于内部权限，需要直属主管确认。
- 临时权限默认 7 天过期，到期自动回收。

## Finance-Reimbursement-SOP / Travel-Policy

差旅报销规则：

- 发票抬头必须使用公司工商全称。
- 税号以财务系统内当前法人主体为准。
- 高铁票、酒店发票、拜访记录、出差审批单需要一起上传。
- 差旅结束后 30 个自然日内提交报销。
- 跨月费用尽量在当月结账日前提交。

## Security-Access-Policy / Database-Access-SOP

生产权限规则：

- 外包人员默认不能直接访问生产数据库。
- 优先使用内部员工代查、日志平台、脱敏查询平台。
- 如确需访问，需要限定工单编号、库表范围、时间窗口、只读权限。
- 审批链路为项目负责人、安全负责人、系统 Owner。
- 所有生产访问必须记录审计日志，到期自动回收。

## Security-Data-Export-Policy / PII-Desensitization-SOP

数据导出规则：

- 不能通过普通邮件发送全量原始日志。
- 包含手机号、用户 ID、IP、设备号等字段时，需要做脱敏或哈希化。
- 数据导出遵循最小必要原则，只提供审计目标需要的字段。
- 审批链路为业务负责人、安全负责人、法务/隐私合规。
- 交付方式应使用受控下载链接，设置有效期、水印和访问审计。

## Sales-Pricing-Playbook / Pilot-Contract-SOP

销售试点报价规则：

- 试点报价需要同时看账号数、试点周期、转年框承诺和 API 调用量。
- 低于标准折扣线需要区域负责人审批。
- 试点转正式年框时，需要在合同中写清转正条件和抵扣规则。
- 不建议口头承诺最低价，必须输出正式报价单并走商务审批。

## 如何作为面试数据集

这些制度片段可以上传到知识库，也可以作为演示文档讲解：

- 每个工单都能映射到一个知识域。
- 每个答复都有明确引用来源。
- 每个场景都有业务风险，而不是简单 FAQ。
- 采纳样本可以进入 Golden Pair。
- 驳回样本可以进入知识缺口，推动知识库更新。
