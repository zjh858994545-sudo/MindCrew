# MindCrew 技术文档：文档入库与检索流程

## 文档目标

本文档聚焦 MindCrew 的知识入库与检索链路，说明一份文档从上传到可问答的完整过程，以及系统如何将多路检索结果转化为带来源的最终答案。

## 一、文档入库流程

### 1. 上传阶段

用户上传 PDF、DOCX、Markdown 或纯文本文件后，系统会先完成以下动作：

- 生成知识库记录
- 保存原始文件到对象存储
- 记录文件类型、大小、上传用户和当前状态

知识库初始状态通常为：

- `uploading`
- `processing`
- `ready`
- `error`

### 2. 文本抽取阶段

系统根据文件类型使用不同的提取策略：

- PDF：抽取页面文本
- Word：读取段落和结构内容
- Markdown：保留标题与自然段落
- TXT：直接按文本读取

文本抽取的目标不是保留原文件排版，而是尽量产出适合检索的纯文本内容。

### 3. 文本切片阶段

抽取后的文本会进入切片器，切片策略通常包括：

- 按段落初步切分
- 过短段落合并
- 过长段落按句子拆分
- 使用固定长度窗口和少量重叠区间

切片需要兼顾两个目标：

- 每个片段要足够短，便于向量检索和模型上下文控制
- 每个片段又要保留完整语义，避免检索结果失真

### 4. 元数据构建阶段

每个切片会附带基础元数据，例如：

- 知识库 ID
- 分类
- 章节
- 页码
- 切片顺序
- 内容类型

这些元数据可用于：

- 检索时过滤
- 最终回答时展示来源
- 后续数据分析与治理

### 5. 向量化与入库阶段

系统将每个切片送入 Embedding 模型，得到向量后写入 Milvus。与此同时，切片原文和元数据仍会保存到 MySQL。

最终形成两套互补索引：

- 向量索引：支持语义检索
- 文本索引：支持关键词检索

## 二、查询检索流程

### 1. Query Rewrite

用户输入通常带有口语化、省略或上下文依赖。系统会在检索前尝试做查询改写，使问题更适合搜索。

例如：

- “这个接口怎么接” 可能改写成 “API 接口接入步骤与认证配置”
- “刚才那个规则还有例外吗” 可能保留原上下文关键词并补充主语

### 2. 向量检索

向量检索适用于：

- 概念解释
- 语义相近表达
- 用户问题和原文描述不完全一致的情况

向量检索的优势是召回面广，但可能带来语义相关而非精确匹配的结果。

### 3. 关键词检索

关键词检索适用于：

- 编号
- 术语
- 条款
- 精确短语
- 专有名词

关键词检索能弥补向量检索在精确定位上的不足，尤其适合查规范、配置项、字段名和固定表述。

### 4. 多路融合

系统不会盲信某一路结果，而是将向量检索和关键词检索召回结果统一融合。常见做法是：

- 按各自排序结果计算融合分
- 对重复内容去重
- 形成统一候选集

这样做的意义在于：

- 保留语义召回能力
- 补强精确匹配能力
- 降低单一路径召回偏差

### 5. 重排与压缩

融合结果进入 Reranker 后，会根据问题和候选内容的相关性进一步排序。之后系统会对结果做上下文压缩，只保留最适合进入 Prompt 的切片。

这个阶段的目标是：

- 提升最终答案质量
- 控制上下文长度
- 保证引用来源足够集中且相关

## 三、来源组装与回答生成

系统会把最终进入 Prompt 的结果统一编号，例如：

- `[1] 知识库《部署手册》 - 环境准备`
- `[2] 网页《官方发布说明》 https://example.com/release`

最终回答阶段要求模型：

- 只能使用提供的来源作答
- 不得伪造出处或链接
- 在结尾标注参考来源

## 四、当检索结果不足时

如果系统发现没有足够可靠的来源，会触发兜底逻辑：

- 明确说明当前知识库结果不足
- 输出通用说明性回答
- 提醒用户补充范围、关键词或相关知识库

这种处理比“强行回答但没有依据”更适合知识库场景。

## 五、影响检索质量的关键因素

### 文档质量

- 原始文档排版清晰
- 标题层次明确
- 避免扫描乱码

### 切片策略

- 切片过长会降低定位精度
- 切片过短会破坏上下文

### 元数据质量

- 分类准确
- 章节和页码可追溯
- 知识库归属清晰

### 查询表达

- 用户问题越明确，召回越稳定
- 包含术语、对象、范围、时间条件时效果通常更好

## 六、实践建议

- 建议按主题、业务域或产品模块拆分知识库
- 同一知识库内保持内容风格相近
- 对频繁问答的文档做标题和章节清洗
- 先导入结构化较好的文档，再逐步补充杂项资料

## 总结

MindCrew 的文档入库与检索流程本质上是“文档治理 + 多路召回 + 来源增强生成”。只要文档结构和分类质量可控，系统就能在保证可追溯性的前提下，显著提升知识问答效率。
