# 企业文档：部署与运维运行手册

## 文档目标

本文档描述一个知识库问答系统在企业环境中的部署、启动、验证、备份和回滚建议，适合作为测试环境和生产环境的运行手册。

## 一、环境分层建议

### 1. 开发环境

用途：

- 本地联调
- 功能验证
- 快速调试

特点：

- 可使用单机 MySQL、Redis、MinIO、Milvus
- 可用 Docker Compose 快速启动

### 2. 测试环境

用途：

- 集成测试
- 回归测试
- 发布前验证

特点：

- 配置尽量接近生产
- 使用独立数据与对象存储空间

### 3. 生产环境

用途：

- 正式服务

建议：

- 分离存储和应用
- 配置备份与监控
- 明确回滚路径

## 二、核心依赖

典型依赖包括：

- MySQL
- Redis
- MinIO
- Milvus
- 聊天模型服务
- Embedding 模型服务

如果启用网页搜索，还需要额外准备外部搜索服务的配置。

## 三、推荐启动顺序

建议按以下顺序启动：

1. MySQL
2. Redis
3. MinIO
4. Milvus
5. 应用服务

原因是应用启动时通常会：

- 读取数据库配置
- 初始化缓存连接
- 校验对象存储
- 初始化向量集合

## 四、上线前检查清单

### 基础检查

- 配置文件是否指向正确环境
- 密钥是否已注入
- 数据库连接是否正常
- MinIO Bucket 是否存在
- Milvus Collection 是否可访问

### 服务检查

- 登录接口是否正常
- 知识库上传是否成功
- 检索接口是否可返回数据
- SSE 问答是否正常
- MCP 工具是否已注册

## 五、备份建议

### MySQL

- 每日全量备份
- 必要时配合 binlog 增量恢复

### MinIO

- 对原始知识文档做定时备份
- 保留关键版本快照

### 配置

- 对环境变量和部署清单做版本化管理

## 六、回滚策略

### 应用层回滚

适用于：

- 新版本代码异常
- 性能显著下降
- 核心功能回归

做法：

- 切回上一个稳定镜像或构建版本

### 数据层回滚

适用于：

- 配置误改
- 数据误删
- 索引异常重建失败

做法：

- 恢复数据库备份
- 恢复对象存储
- 重建向量索引

## 七、日常巡检项

建议每日或每次发布后关注：

- 应用是否正常启动
- 数据库连接池是否稳定
- 缓存是否异常膨胀
- 向量检索耗时是否异常
- 文档处理失败率是否升高
- 问答接口响应时间是否超出阈值

## 八、常见故障排查

### 场景一：知识库上传成功但无法检索

优先排查：

- 文档是否完成切片
- 向量是否成功写入 Milvus
- 知识库状态是否为 `ready`

### 场景二：问答返回兜底结果过多

优先排查：

- 知识库是否为空
- Query Rewriter 是否异常
- 检索参数是否过严
- Embedding 或 Reranker 服务是否异常

### 场景三：MCP 工具不可见

优先排查：

- MCP 配置是否启用
- 工具 Bean 是否注册
- 鉴权是否阻止访问 `/mcp`

## 九、总结

运行一个稳定的知识库问答系统，关键不只是把应用启动起来，而是建立完整的“部署、验证、巡检、备份、回滚”闭环。运行手册的价值在于把这些步骤标准化，降低对单个研发人员经验的依赖。
