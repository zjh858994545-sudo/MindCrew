# MindCrew 技术文档：系统总体架构

## 文档目标

本文档说明 MindCrew 作为通用知识库检索与问答系统的总体技术架构，帮助研发、实施和运维人员理解系统边界、核心模块、运行依赖以及主要数据流。

## 系统定位

MindCrew 是一个面向多领域知识管理场景的智能检索系统。系统支持文档上传、切片建库、向量检索、关键词检索、Agentic RAG 问答以及 MCP 工具对外暴露。它既可以作为单体 Web 应用提供页面和 API 服务，也可以作为知识能力底座，为外部 Agent 或自动化流程提供可调用的工具接口。

## 核心组件

### 1. Web 应用层

- 基于 Spring Boot 提供 REST API、SSE 流式问答接口和后台管理能力。
- 前端负责知识库管理、会话展示、消息流渲染和来源展示。
- 安全层使用 JWT 做认证，部分接口支持通过 Header 或 URL 参数传递令牌。

### 2. 数据存储层

- MySQL：保存用户、知识库、文档切片、会话、消息和工具注册信息。
- Redis：保存缓存、频次统计以及用户长期记忆。
- MinIO：保存上传的原始文档与附件文件。
- Milvus：保存向量数据，用于语义检索。

### 3. 智能处理层

- Embedding 模型：将查询和文档切片映射为向量。
- Query Rewriter：将口语化问题改写为更适合检索的表达。
- Vector Retriever：从 Milvus 中按语义相似度召回候选切片。
- BM25 Retriever：从 MySQL 中按关键词召回候选切片。
- RRF Fusion：融合多路召回结果。
- Reranker：对融合后的结果做更精细的排序。
- Prompt Assembler：把来源、历史、长期记忆组装为最终提示词。
- Self Reflection：对回答质量做二次审查。

### 4. 工具与集成层

- 内部工具：`doc_search`、`keyword_search`、`web_search`、`recall_memory`、`store_memory`
- MCP Server：将工具能力暴露给外部客户端
- Web Search：可选接入外部搜索服务，为时效性问题补充网页结果

## 一次问答的典型流程

1. 用户通过前端或 API 提交问题
2. 系统识别意图并选择工具
3. 对问题做查询改写
4. 调用向量检索、关键词检索或网页检索
5. 对召回结果做融合与重排
6. 组装 Prompt，注入历史和长期记忆
7. 调用大模型流式生成答案
8. 对结果做自检与兜底处理
9. 持久化消息、来源、工具调用日志和推理链
10. 通过 SSE 持续向前端发送事件与最终结果

## 关键设计原则

### 单体优先

MindCrew 当前采用单体应用形态，原因是：

- 业务链路清晰，功能边界集中
- 研发与调试成本低
- 更适合课程设计、内部项目和中小规模业务快速迭代

### 检索与生成解耦

系统将“检索候选内容”和“生成最终答案”拆分为两个阶段。这样做可以让检索路径更容易评估，也便于在生成阶段附带明确来源。

### 内部直调，外部走 MCP

- 应用内部直接调用 Spring Bean 形式的工具实现
- 外部 Agent 或 MCP Client 通过 `/mcp` 调用同一批工具能力

这种设计避免了应用内部“通过 MCP 再调用自己”的额外复杂度，同时保证工具定义与外部暴露接口保持一致。

## 主要数据对象

### 知识库对象

- 知识库名称
- 分类
- 描述
- 文件地址
- 状态
- 切片数量

### 文档切片对象

- 所属知识库 ID
- 文本内容
- 切片顺序
- 元数据
- 对应向量 ID

### 会话与消息对象

- 会话标题
- 关联知识库 ID 列表
- 用户问题与助手回答
- 来源列表
- MCP 调用记录
- 推理链与自检日志

## 外部依赖

部署一个完整的 MindCrew 环境通常至少需要以下组件：

- MySQL 8
- Redis 7
- MinIO
- Milvus 2.x
- 一个兼容 OpenAI 协议的聊天模型
- 一个可用的 Embedding 模型

如果启用网页搜索，还需要额外配置外部搜索服务的地址与密钥。

## 可观测性

系统当前主要通过应用日志和数据库记录进行排查：

- 检索链路日志
- 工具调用日志
- SSE 事件输出
- MCP 工具注册与调用统计

建议后续补充：

- 接口耗时指标

- 检索命中率指标
- 大模型调用成本指标

## 风险与限制

- 当知识库为空时，系统只能走通用兜底回答
- 当外部搜索未配置时，`web_search` 会自动降级为空结果
- 当 Milvus 或 Embedding 模型异常时，向量检索链路会受影响
- 当知识库文档质量差、切片不合理或分类混乱时，问答效果会明显下降

## 总结

MindCrew 的架构适合构建一个“可解释、可追溯、可扩展”的知识库问答系统。它将文档管理、检索增强生成、MCP 工具暴露和用户交互放在统一工程中，便于后续继续演进为多租户、分布式或更强 Agent 能力的平台。
