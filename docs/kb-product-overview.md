# MindCrew 产品说明：产品概览

## 产品定位

MindCrew 是一个面向企业团队、项目组和知识管理场景的通用知识库检索系统。它帮助用户把分散的规范文档、产品说明、技术手册、FAQ、流程文档和外部说明资料组织成可检索、可问答、可追溯的知识资产。

## 目标用户

### 1. 团队成员

需要快速查资料、看规范、找流程，但不想反复翻文档。

### 2. 产品经理

希望沉淀产品说明、版本规则、业务流程和功能边界，降低重复沟通成本。

### 3. 研发与测试人员

需要快速定位接口规范、部署说明、错误处理策略、配置项含义和历史约束。

### 4. 知识管理或运营人员

需要统一管理知识库、维护文档版本、跟踪常见问题和优化问答质量。

## 产品价值

### 降低查找成本

相比手动翻目录或全文搜索，问答式检索更适合复杂问题、组合问题和模糊表达。

### 提升知识复用率

把“文档存在，但没人用”“信息在群里反复问”转化为可复用知识资产。

### 保证回答可追溯

系统不仅给答案，还会附上知识来源，避免无依据回答。

### 支持外部能力集成

通过 MCP 暴露工具后，知识库能力可以被外部 Agent 或自动化流程复用。

## 核心功能

### 文档知识库管理

- 上传文档
- 查看处理状态
- 删除或维护知识库
- 按分类组织内容

### 智能问答

- 支持普通问答
- 支持追问
- 支持指定知识库范围
- 支持带来源的回答

### 会话管理

- 保留历史对话
- 支持会话列表
- 支持消息反馈
- 支持导出 Markdown

### 工具开放

- 支持 MCP Tool 暴露
- 支持语义检索、关键词检索、网页检索和长期记忆能力

## 典型使用场景

### 场景一：项目成员快速查规范

用户不需要打开多个文档文件，只要直接提问：

- “角色权限继承有哪些限制？”
- “测试环境部署需要哪些前置条件？”

### 场景二：产品资料统一检索

将 PRD、功能说明、版本记录、FAQ 导入后，团队成员可以统一提问：

- “这个功能为什么要加审批节点？”
- “退款规则在新版里有什么变化？”

### 场景三：外部 Agent 调用知识库能力

当外部自动化系统需要访问企业知识时，可以通过 MCP 调用：

- 检索内部文档
- 获取网页补充资料
- 结合长期偏好输出结果

## 产品边界

MindCrew 适合做：

- 企业内部知识管理
- 文档问答
- 结构化来源引用
- 工具化知识服务

MindCrew 不适合直接替代：

- 复杂工作流引擎
- 正式决策审批系统
- 全功能文档协作平台
- 强事务型业务系统

## 成功使用的关键条件

- 文档质量较好
- 分类和命名清晰
- 团队愿意持续维护知识库
- 重要知识以文档形式沉淀，而不是只存在口头沟通里

## 总结

MindCrew 的核心目标不是“生成更多内容”，而是“让已有知识更容易被找到、理解和复用”。它更适合作为团队的知识检索中枢，而不是泛化的聊天工具。
