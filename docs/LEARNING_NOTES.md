# 学习笔记

## RAG 工程化

- RAG 的核心不只是向量检索，而是召回、排序、引用、评测和反馈的系统工程。
- 向量召回适合语义相近问题，BM25 对专有名词、编号、关键词更稳。
- RRF 可以在不知道各召回器分数是否同尺度的情况下融合排名。
- Rerank 适合把候选结果再精排，但需要控制候选数量和耗时。

## Agent 工程化

- Agent 的可解释性来自 Trace，而不是只打印日志。
- 每一步都应记录输入摘要、输出摘要、耗时、状态和错误。
- 可观测性系统要可降级，不能因为 Trace 写入失败影响主问答。

## 安全工程

- Prompt Injection 不只来自用户，也可能来自被检索文档。
- 最终答案输出前仍要做脱敏，因为模型可能复述上下文中的敏感内容。
- 安全事件应该可审计，但日志里不能保留真实密钥。

## 评测工程

- 评测需要固定数据集、固定指标、固定策略配置。
- 离线确定性评测适合作为 CI 基线。
- 线上真实模型评测适合做版本趋势和质量监控。

## 面试准备

- 不要只说“用了大模型”，要说清楚链路、指标、取舍和问题定位方式。
- 项目亮点最好能落到页面、接口、SQL、测试、文档五类证据上。
