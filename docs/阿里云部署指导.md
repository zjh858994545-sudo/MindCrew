# 阿里云生产部署指南

> 目标：把 DocMind / MindCrew 从本地开发环境完整搬到阿里云，支持 500 并发用户、视频/音频/图片全模态问答、跨厂商模型切换。

---

## 一、阿里云资源清单

| 资源 | 规格建议 | 用途 | 备注 |
|---|---|---|---|
| **ECS · 应用服务器** | 8 核 16G（ecs.c7.2xlarge）× 2 台 | Spring Boot 应用 + Nginx | 起步可单台；500 并发上 2 台做负载均衡 |
| **ECS · 中间件服务器** | 8 核 32G（ecs.r7.2xlarge）× 1 台 | Milvus + Redis（可合并）| 也可全部上托管服务 |
| **RDS MySQL** | 8 核 16G，500GB SSD | 业务数据库 | 选 MySQL 8.0；开 binlog 备份 |
| **OSS Bucket** | 标准存储，公共读 | 文档/音视频原文件存储 | **必须**：DashScope ASR 拉音轨要走公网 URL |
| **CDN**（可选） | 与 OSS 绑定 | 加速文件下载 / 视频播放 | 大量视频时显著提升体验 |
| **SLB · 负载均衡** | 性能保障型 slb.s2.small | 多 ECS 实例的入口 | 配 HTTPS 证书 |
| **域名 + SSL 证书** | 阿里云免费证书即可 | HTTPS 接入 | DashScope 等回调对 HTTP 不友好 |
| **DashScope（百炼）** | 按 token 计费 | LLM / VL / ASR / Embedding | 入口：`https://dashscope.aliyun.com` |
| **NAS / EFS**（可选） | 100GB | 多 ECS 间共享上传临时文件 | 单机不用，多机才需要 |

### 不在阿里云买的部分

| 服务 | 来源 | 说明 |
|---|---|---|
| Milvus | 自部署在 ECS 上（Docker） | 阿里云没有官方 Milvus 服务；想偷懒可用 Zilliz Cloud |
| Tavily Web 搜索 | tavily.com | 联网搜索 API，按调用计费 |

## 二、系统依赖（必装）

应用 ECS 上必须装这几个二进制，否则部分功能会失败：

```bash
# 基础
sudo apt update
sudo apt install -y openjdk-17-jdk nginx

# 视频/音频处理（视频上传必装！）
sudo apt install -y ffmpeg
ffmpeg -version    # 应输出 7.x 或 6.x

# Office 文档转换（WPS/老 doc 格式需要）
sudo apt install -y libreoffice
soffice --version  # 应输出 LibreOffice 7.x

# 中文字体（视频画面 OCR、PDF 渲染会用到）
sudo apt install -y fonts-noto-cjk fonts-wqy-zenhei
```

> 推荐用 Docker 部署，把这些依赖都打进镜像，免得每台 ECS 都装一遍。Dockerfile 见第六节。

## 四、OSS Bucket 配置

这是 ASR / 视频功能能跑通的**关键**——本地 MinIO 走 localhost URL，DashScope 拉不到；OSS 公网 URL 才能让 ASR 工作。

### 1. 创建 Bucket

阿里云控制台 → 对象存储 OSS → 创建 Bucket：

| 项 | 值 |
|---|---|
| Bucket 名称 | `mindcrew-prod`（全局唯一） |
| 地域 | 跟 ECS 同地域，如华东 1（杭州） |
| 存储类型 | 标准存储 |
| 读写权限 | **公共读**（让 DashScope 能拉文件） |
| 服务端加密 | OSS 完全托管（KMS） |
| 版本控制 | 关闭（节省成本） |

> ⚠️ "公共读"只代表**有 URL 就能读**，不是文件列表公开。生产更安全的做法是开"私有"+ 让代码生成预签名 URL（默认 7 天有效），代码已经实现这套逻辑。

### 2. RAM 子账号授权

不要直接用主账号 AK：

1. 控制台 → 访问控制 RAM → 创建用户 `docmind-app`
2. 给用户加 AccessKey
3. 授权策略：`AliyunOSSFullAccess`（或自定义只授权这一个 Bucket，更严）

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["oss:*"],
      "Resource": [
        "acs:oss:*:*:mindcrew-prod",
        "acs:oss:*:*:mindcrew-prod/*"
      ]
    }
  ]
}
```

### 3. 验证 OSS 通畅

```bash
# 在 ECS 上跑一次
curl -X PUT "https://mindcrew-prod.oss-cn-hangzhou.aliyuncs.com/test.txt" \
  -H "Authorization: ..." \
  --data "hello"
# 200 OK 即通
```

## 五、Milvus 部署

阿里云没有原生 Milvus 服务，推荐 Docker 方式自部署。

### 1. 单机版（起步够用）

```bash
# 在中间件 ECS 上跑
mkdir -p /data/milvus
cd /data/milvus

wget https://github.com/milvus-io/milvus/releases/download/v2.4.0/milvus-standalone-docker-compose.yml -O docker-compose.yml

docker-compose up -d
```

验证：

```bash
# Milvus 服务端口 19530
curl -X POST http://localhost:9091/api/v1/health
# {"status":"OK"}
```

### 2. 集群版（500+ 并发推荐）

```bash
# 用 Helm 装到 Kubernetes（阿里云 ACK）
helm repo add milvus https://zilliztech.github.io/milvus-helm/
helm install my-milvus milvus/milvus --set cluster.enabled=true --set queryNode.replicas=3
```

### 3. 偷懒方案：Zilliz Cloud

直接买 Zilliz Cloud（Milvus 官方托管），免运维：
- 入口：https://cloud.zilliz.com
- 在 application-prod.yml 改成 Zilliz 的连接串和 Token 即可

## 六、Docker 部署

### Dockerfile

放在项目根目录：

```dockerfile
FROM openjdk:17-jdk-slim

# ── 系统依赖（视频/音频/Office 必装）──
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        ffmpeg \
        libreoffice \
        fonts-noto-cjk \
        fonts-wqy-zenhei \
        curl && \
    rm -rf /var/lib/apt/lists/*

# ── 时区设置 ──
ENV TZ=Asia/Shanghai
RUN ln -sf /usr/share/zoneinfo/$TZ /etc/localtime

WORKDIR /app
COPY target/MindCrew-*.jar /app/MindCrew.jar

# 启动参数
ENV JAVA_OPTS="-Xms2g -Xmx4g -XX:+UseG1GC -Dfile.encoding=UTF-8"
ENV SPRING_PROFILES_ACTIVE=prod

EXPOSE 8080
HEALTHCHECK --interval=30s --timeout=10s --retries=3 \
    CMD curl -f http://localhost:8080/actuator/health || exit 1

ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar /app/MindCrew.jar"]
```

### docker-compose.prod.yml

```yaml
version: '3.8'
services:
  mindcrew-app:
    build: .

      - "8080:8080"

      BAILIAN_API_KEY: ${BAILIAN_API_KEY}

        reservations:
          memory: 2G
```

启动：

```bash
docker-compose -f docker-compose.prod.yml up -d
docker-compose logs -f mindcrew-app
```

## 八、前端部署

前端是纯静态资源，建议**单独**部署到 OSS+CDN 或 ECS 上的 Nginx。

### 方案 A：Nginx 同机部署（简单）

```bash
# 构建
cd DocMind-frontend
npm install
npm run build       # 产物在 dist/

# 部署到 Nginx
sudo cp -r dist/* /var/www/mindcrew/
```

Nginx 配置 `/etc/nginx/sites-available/mindcrew.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate     /etc/nginx/ssl/your-domain.com.pem;
    ssl_certificate_key /etc/nginx/ssl/your-domain.com.key;

    # 前端静态资源
    root /var/www/mindcrew;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 流式响应必加
        proxy_buffering off;
        proxy_read_timeout 600s;
        chunked_transfer_encoding on;
    }

    # 文件上传大小（视频文件可能很大）
    client_max_body_size 500M;
}
```

### 方案 B：OSS + CDN 部署（高性能）

```bash
# 构建后 sync 到 OSS
ossutil cp -r dist/ oss://mindcrew-frontend/ --update

# OSS 控制台：开静态网站托管 → 设默认首页 index.html / 默认 404 -> index.html（SPA 需要）
```

然后绑定 CDN 域名，加 HTTPS 证书。后端 API 仍走 ECS 的域名，前端通过 CORS 调过去（或者 CDN 配置回源规则）。

## 九、常见坑（实战踩过的）

### 1. ASR 永远失败

**症状**：上传视频/音频，处理完成 → 没有任何识别结果，errorMsg 写着 ASR 失败

**原因**：`storage.type=minio` 或 OSS 配错，DashScope 拉不到音轨 URL

**排查**：
```bash
# 看 application-prod.yml 里 storage.type 必须是 oss
# 看后端启动日志，应该有：
[Storage] 使用 OSS · bucket=mindcrew-prod · endpoint=https://oss-cn-hangzhou.aliyuncs.com
```

### 2. ffmpeg 找不到

**症状**：上传视频报错 `Cannot run program "ffmpeg": error=2, No such file or directory`

**原因**：系统没装 ffmpeg

**修复**：`apt install ffmpeg` 或确保 Docker 镜像里有

### 3. LibreOffice 转换超时

**症状**：上传 .wps / 老 .doc / .ppt 失败

**原因**：libreoffice 没装或第一次转换需要"预热"

**修复**：装好 libreoffice 后**先执行一次**任意转换预热：
```bash
echo "test" > /tmp/test.txt
soffice --headless --convert-to docx /tmp/test.txt --outdir /tmp/
```

### 4. Milvus 写入卡死

**症状**：文档处理到"向量化"那一步卡住、超时

**原因**：Milvus 单机版内存不够，或者 Embedding 维度对不上（必须 1024）

**修复**：
- Milvus 所在 ECS 至少 16GB 内存
- 确认 `milvus.dimension: 1024` 跟 `text-embedding-v3` 一致

### 5. Redis 连接数爆掉

**症状**：500 并发时报 `Could not get a resource from the pool`

**修复**：把 `lettuce.pool.max-active` 调到 50+，或换 Redis 集群

### 6. 上传大视频 500 错误

**症状**：上传 500MB+ 视频，前端报 500 或 413

**修复**：
- Nginx：`client_max_body_size 1G;`
- Spring：`spring.servlet.multipart.max-file-size: 1GB`

### 7. SSE 流式响应被 Nginx 缓存

**症状**：问答接口前端等很久才一次性出答案（应该流式逐字出）

**修复**：Nginx location 加上 `proxy_buffering off;`（第八节配置里已有）
