# MindCrew 快速部署

一键拉起：前端 + 后端 + MySQL / Redis / Milvus / MinIO / etcd。
跟着下面的命令走即可，每步都有说明。

> 详细排错见 [DEPLOY.md](./DEPLOY.md)。

---

## 0. 前置

- Docker 20.10+ 和 `docker compose` v2（`docker compose version` 能输出即可）
- 内存 ≥ 8G（Milvus + JVM 比较吃）
- 服务器安全组放行 **80** 端口（不放行浏览器打不开）

**国内服务器先配镜像加速器**（否则拉 Milvus/MinIO 镜像会超时）：

```bash
# 编辑 /etc/docker/daemon.json，写入：
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ]
}
# 保存后重启 Docker
sudo systemctl daemon-reload && sudo systemctl restart docker
```

---

## 1. 拿代码

```bash
git clone <仓库地址> && cd MindCrew/MindCrew     # 进到与 docker-compose.yml 同级的目录
```

> 已经 clone 过的，确保是最新代码：`git fetch && git reset --hard origin/<分支>`

---

## 2. 配环境变量

```bash
cp .env.docker.example .env
vim .env
```

**只有一个必填**：

```ini
BAILIAN_API_KEY=sk-xxxx        # 阿里云百炼，https://bailian.console.aliyun.com
```

生产再改这几个（开发可跳过）：`MYSQL_ROOT_PASSWORD`、`CRYPTO_MASTER_KEY`、`JWT_SECRET`（用 `openssl rand -base64 48` 生成）。

> 用阿里云 OSS 存文件：把 `STORAGE_TYPE=oss` 并填 `OSS_*`；否则默认用内置 MinIO，不用管。

---

## 3. 启动

```bash
docker compose up -d --build
```

首次约 5~15 分钟：构建前后端镜像 → 拉中间件 → MySQL 建表 → 启动后端/前端。

---

## 4. 验证

```bash
# 4.1 所有容器都该是 Up (healthy)，minio-init 是 Exited (0)
docker compose ps

# 4.2 后端起来了：看到 "Started MindCrewApplication in X seconds" 即成功
docker compose logs backend | grep -iE 'Started MindCrew|ERROR' | tail -5

# 4.3 建表完整、无乱码
docker exec mindcrew-mysql mysql --default-character-set=utf8mb4 \
  -uroot -p"$MYSQL_ROOT_PASSWORD" docmind -e "SELECT name FROM system_persona;"
```

后端 healthy 后，浏览器打开 **`http://<服务器IP>/`**（80 端口，**不是** 8080，更不是 22）。

---

## 5. 更新代码后重新部署

```bash
git fetch && git reset --hard origin/<分支>     # 拉最新（会覆盖本地改动）
docker compose up -d --build                    # 重建并滚动更新
```

> 只动了前端：`docker compose up -d --build frontend`
> 只动了后端：`docker compose up -d --build backend`
> 改了 `sql/` 或 `init.sh`：这俩是挂载进去的，不用重建镜像，但要重置 MySQL 才会重新建表（见下）。

## 6. 常见问题

| 现象 | 处理 |
|---|---|
| 浏览器打不开 | 用 **80** 端口 `http://IP/`；检查云安全组放行 80 |
| 拉镜像超时 | 配镜像加速器（第 0 步） |
| 前端 502 | 后端没起好，`docker compose logs backend` 看报错 |
| 后端报 `Table 'docmind.xxx' doesn't exist` | 建表没跑全，重置 MySQL（下方命令） |
| 页面中文乱码 `ä¸­ç«` | 导入字符集问题，重置 MySQL 重新导入 |
| 端口被占 | 改 `.env` 里 `*_PORT`，重新 `up -d` |

**重置 MySQL（清库重新建表，⚠️ 业务数据会清空）：**

```bash
docker compose rm -sf mysql backend
docker volume rm mindcrew_mindcrew_mysql_data     # 注意是双 mindcrew 前缀
docker compose up -d
```

**定位建表是哪一步挂的：**

```bash
docker logs mindcrew-mysql 2>&1 | grep -iE 'mindcrew-init|ERROR'
# 打印 [mindcrew-init] -> 文件名，停在哪个文件就是那个 SQL 有问题
```

## 常用运维

```bash
docker compose down            # 停服务，保留数据
docker compose down -v         # 停服务并清空所有数据（⚠️ 慎用）
docker compose restart backend # 重启单个服务
docker compose logs -f backend # 实时看日志
```
